# 성장 시스템 기획 — 유물 / 어빌리티 / 장비

> 작성일: 2026-04-26  
> 회귀(런) 구조: 어빌리티·장비는 런마다 초기화, 유물은 영구 유지

---

## 1. 유물 (Relic) — 영구 성장

### 개요
- 장군 **공통** 적용 — 보유한 모든 장군에게 동일하게 적용
- 초반 영향 미미, 누적될수록 강력한 복리 효과
- % 수치 증가 (절대값 아님)

### 능력치 종류
| 능력치 | 설명 |
|--------|------|
| 공격력 | 모든 장군 ATK % 증가 |
| 공격속도 | 모든 장군 Attack Speed % 증가 |
| 이동속도 | 모든 장군 Move Speed % 증가 |
| 보유 병사 수 | 장군이 소환·지휘 가능한 병사 최대 수 증가 |
| 병사 지휘력 | 병사 스탯(ATK/HP 등) % 증가 |
| (추가 예정) | 방어율, 체력, 스킬 쿨다운 등 |

### 등급 & 강화 방식
- 중복 획득 → **레벨 업** (수치 누적 상승)
- 레벨업에 필요한 중복 갯수는 레벨이 오를수록 증가 (무한 강화)
  - 예: Lv1→2: 2개, Lv2→3: 4개, Lv3→4: 8개 …

### 획득 방식
| 방법 | 내용 |
|------|------|
| 젬 가챠 | 젬 소모 → 랜덤 유물 획득 (중복 시 레벨업) |
| 특수 던전 | 던전 클리어 보상으로 특정 유물 파밍 가능 (중복 파밍 시 레벨업) |

## 2. 어빌리티 (Ability) — 런 내 성장

### 개요
- 스테이지 **회귀(런 종료) 시 초기화**
- 등급이 있으며 특수 어빌리티는 전략 자체를 바꿀 수 있음
- % 수치 증가 (절대값 아님)

### 등급 구조
| 등급 | 특성 |
|------|------|
| 일반 | 기본 능력치 증가 (공격력, 공속, HP 등) |
| 고급 | 더 높은 % 수치 증가 |
| 특수 | 고유 효과 — 1개만 보유 가능, 전략적 변화 유발 |

### 중복 처리
- **일반/고급 어빌리티**: 동일 어빌리티 중복 등장 가능 → 수치 **합연산** 누적
- **특수 어빌리티**: 1개만 보유, 중복 선택 불가

### 획득 방식
- 스테이지 클리어 후 **3택지** 중 1개 선택
- 택지 구성은 현재 런 진행도·등급 가중치에 따라 랜덤

### 특수 어빌리티 예시 방향
- 스킬 연계 효과 ("HeavyStrike 사용 후 다음 공격 치명타 100%")
- 병사 변환 ("모든 병사가 돌격 특성 획득")
- 조건부 폭발 효과 ("적 처치 시 인근 폭발")

## 3. 장비 (Equipment) — 런 내 성장

### 개요
- 스테이지 **회귀(런 종료) 시 초기화**
- **절대값** 증가 (유물·어빌리티의 % 증가와 차별화)
- 부족한 능력치 보완 또는 강점 증폭에 활용

### 슬롯
- 장군 1인당 **2개** 장착 가능

### 등급 구조
| 등급 | 특성 |
|------|------|
| 일반 | 기본 스탯 절대값 증가 (ATK +N, HP +N 등) |
| 고급 | 더 높은 절대값 |
| 상위 | 절대값 + **조건부 효과** (공격 시 / 피격 시 발동) |

### 강화
- **강화석 + 골드** 소모로 강화 가능
- 강화 단계마다 수치 추가 상승
- 강화는 런 내에서만 유효 (회귀 시 초기화)

### 획득 방식
- 스테이지 클리어 시 **스테이지 레벨에 따라** 렌덤 장비 1개 선택지 제공
  - 낮은 스테이지: 일반 등급 위주
  - 높은 스테이지: 고급·상위 등급 등장 확률 증가

## 시스템 비교 요약

| 항목 | 유물 | 어빌리티 | 장비 |
|------|------|---------|------|
| 지속성 | **영구** | 런 초기화 | 런 초기화 |
| 증가 방식 | % | % | **절대값** |
| 슬롯 제한 | 없음 (누적) | 특수만 1개 제한 | 2개 |
| 획득 | 젬 가챠·특수 던전 | 스테이지 클리어 3택 | 스테이지 클리어 1택 |
| 강화 | 중복 누적 (레벨업) | 동일 어빌리티 합연산 | 강화석+골드 |
| 전략 영향 | 장기 성장 베이스 | 런 전략 결정 | 수치 세밀 조정 |

## 구현 시 참고

- 유물: `RelicData` SO + `RelicRuntimeSystem` — 현재 `PassiveSkillRuntimeSystem` 패턴 그대로 적용 가능
- 어빌리티: `AbilityData` SO, 런 시작 시 `GeneralComponent`에 누적 적용, 런 종료 시 초기화
- 장비: `EquipmentData` SO, `GeneralComponent`에 슬롯 2개, 절대값은 스탯 계산 최종 단계에서 합산
- 세 시스템 모두 `StatType.cs` 기반으로 수치 적용 → 스탯 계산 우선순위: **유물(%)→ 어빌리티(%)→ 장비(절대값)** 순 권장
